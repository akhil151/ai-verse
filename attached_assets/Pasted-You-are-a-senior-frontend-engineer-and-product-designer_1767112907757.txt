You are a senior frontend engineer and product designer working on a hackathon-winning GenAI product.

PROJECT CONTEXT:
We are building “Nivesh.ai” — a multilingual AI Funding Co-Founder for Indian startup founders.

This is NOT a generic chatbot.
This is a decision-support system that helps founders make correct funding decisions using verified Indian startup funding data, explained in Indic languages.

The backend is already built using FastAPI with a RAG (Retrieval-Augmented Generation) system.
Your job is to build a complete, professional, production-grade frontend that can later be connected to backend APIs.

TARGET USERS:
- Early-stage Indian startup founders
- Non-technical founders
- Users who prefer Indic languages (Hindi, Tamil, Telugu, etc.)

DESIGN GOAL:
- Professional, modern, minimal
- Clean typography
- Subtle animations
- High trust, not playful or childish
- Looks like a serious SaaS product or investor tool

TECH STACK (FRONTEND):
- React + Vite OR Next.js (App Router preferred)
- Tailwind CSS (or equivalent modern styling)
- Component-based architecture
- Clean folder structure
- Ready to connect to FastAPI endpoints later

--------------------------------------------------
CORE FRONTEND FEATURES (MUST IMPLEMENT):

1. LANDING PAGE
- Product name: Nivesh.ai
- Clear one-line value proposition:
  “Your AI Funding Co-Founder — Decisions, not answers.”
- Sections:
  - Problem (funding info is complex & English-heavy)
  - Solution (AI-powered, multilingual, grounded in real data)
  - How it Works (3-step visual explanation)
  - Call to Action: “Start Funding Assessment”

2. FOUNDER ONBOARDING FLOW (CRITICAL)
Multi-step form (stepper UI):
- Startup Stage (Idea / MVP / Revenue / Growth)
- Sector (SaaS, FinTech, AgriTech, HealthTech, etc.)
- Location (State / City)
- Funding Goal (Grant / Angel / VC)
- Preferred Language (English, Hindi, Tamil, Telugu)
Store this context globally (Context API / Zustand / Redux).

This context must visually feel important.

3. MAIN DASHBOARD (CORE SCREEN)
Split layout:
- Left panel: Conversation / Query Input
- Right panel: Intelligence Panel

LEFT PANEL:
- Input box for natural language questions
- Language selector
- Submit button
- Chat-style message history (professional styling)

RIGHT PANEL (KEY DIFFERENTIATOR):
- Funding Readiness Score (0–100 with progress bar)
- Funding Recommendation (Angel / VC / Grant)
- Reasoning Summary (Why this decision)
- Confidence Indicator (High / Medium / Low)

4. ACTIONABLE NEXT STEPS PANEL
Below every AI response, show:
- “Next 7-Day Action Plan”
- Checklist UI (documents, registrations, metrics)
- Clean, readable bullets

5. EXPLAINABILITY & TRUST PANEL
Expandable section:
- Sources used (policy docs / reports)
- Disclaimer: “Based on verified public data”
- Confidence score

6. LANGUAGE SUPPORT UX
- Clear language toggle
- Content direction should adapt correctly
- No hardcoded text — prepare for dynamic backend responses

--------------------------------------------------
EXTRA FEATURES (IMPRESS JUDGES):

- Loading states with skeleton UI
- Subtle transitions & micro-animations
- Empty states that educate users
- Error handling UI (API failure, no data, etc.)
- Responsive layout (desktop-first, tablet friendly)

--------------------------------------------------
DESIGN SYSTEM:
- Neutral professional color palette (dark blue / slate / white)
- Avoid bright or playful colors
- Use icons sparingly
- Consistent spacing and hierarchy
- No clutter

--------------------------------------------------
DELIVERABLES:
- Full frontend codebase
- Clean component structure
- Reusable UI components
- Mock API service layer (fake JSON responses for now)
- Easy-to-connect API abstraction for FastAPI later

--------------------------------------------------
IMPORTANT RULES:
- Do NOT build a childish or cartoon UI
- Do NOT make it look like a chat demo
- Treat this as a serious funding intelligence platform
- Prioritize clarity, trust, and decision support

Build this frontend as if it will be shown to judges, investors, and founders.
